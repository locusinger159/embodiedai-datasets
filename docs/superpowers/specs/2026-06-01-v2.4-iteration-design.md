# v2.4 迭代设计：内容深度化 + 数据标准化基础

> 日期：2026-06-01 | 状态：待审核

## 背景

具身智能数据集导航站（superdata-robotai.com）当前为 v2.3，已收录 55 个数据集和 19 个数据标准，具备详情页、多选对比、changelog、giscus 评论、质量标签、BibTeX 引用等功能。

用户长期目标：
1. 通过导航站建立行业影响力和专家品牌
2. 基于数据集分析积累，提出具身智能数据统一格式标准
3. 标准被采纳后，开发数据生产工具链、标准数据定制等商业化产品

本次迭代（v2.4）定位为**从"目录"向"分析平台"过渡**的关键版本。

## 目标

- **英文版上线**：解锁国际受众，SEO 双语言覆盖
- **内容杠杆建立**：新增技术博客板块，为专家品牌打基础
- **标准地基铺设**：数据格式对比功能 + 标准草案页，为后续标准化工作铺垫

## 模块一：国际化（i18n）

### 设计

保持现有零依赖静态构建架构。采用"平行数据文件 + 平行模板"方案，不引入运行时翻译。

#### 数据层

```
docs/data/
  ├── datasets.json          # 中文数据（已有，不改动）
  ├── datasets.en.json       # 英文数据（新增，字段名一致，值翻译为英文）
  ├── standards.json         # 中文标准（已有）
  └── standards.en.json      # 英文标准（新增）
```

`datasets.en.json` 与 `datasets.json` 的 JSON 结构完全一致，仅字段值翻译为英文。`id` 字段保持一致，用于跨语言关联。

#### 模板层

```
src/pages/
  ├── index.html             # 中文首页
  ├── datasets.html          # 中文数据集列表
  ├── dataset-detail.html    # 中文数据集详情
  ├── standards.html         # 中文标准列表
  ├── standard-detail.html   # 中文标准详情
  ├── submit.html            # 中文提交页
  └── en/                    # 英文模板（新增）
      ├── index.html
      ├── datasets.html
      ├── dataset-detail.html
      ├── standards.html
      ├── standard-detail.html
      └── submit.html
```

模板共享 `{{HEAD}}`、`{{NAVBAR}}`、`{{FOOTER}}` 中的 UI 文案通过构建时的 i18n 映射表注入。

#### UI 字符串

在 `build.cjs` 中新增 `I18N` 常量对象：

```js
const UI = {
  zh: { home: '首页', datasets: '数据集', standards: '数据标准', /* ... */ },
  en: { home: 'Home', datasets: 'Datasets', standards: 'Standards', /* ... */ }
};
```

导航栏和 Footer 根据当前构建语言选择对应的 UI 字符串。

#### 构建输出

```
dist/
  ├── index.html             # 中文首页（默认）
  ├── datasets/
  ├── standards/
  ├── en/                    # 英文站
  │   ├── index.html
  │   ├── datasets/
  │   └── standards/
  └── CNAME
```

#### 语言切换

导航栏右侧增加语言切换按钮：当前语言高亮，点击跳转到对应语言的同页面。

URL 映射规则：
- 中文：`/datasets/open-x-embodiment.html`
- 英文：`/en/datasets/open-x-embodiment.html`

### 数据翻译策略

- 55 个数据集的 `description`、`notes`、`quality.collection`、`quality.annotation` 等文本字段需要翻译
- 推荐用 Claude/DeepL 做初译 + 人工审核修正
- 机构名、URL、技术术语保留原文
- 预计翻译量：每数据集约 200-500 英文词，总计约 15000-25000 词

## 模块二：技术博客板块

### 设计

纯静态博客，Markdown 或 JSON 作为数据源，构建时生成 HTML。

#### 数据源：`docs/data/blog.json`

```json
[
  {
    "id": "av-pipeline-lessons",
    "title": "从自动驾驶数据管线看具身智能数据标准化",
    "titleEn": "Lessons from AV Data Pipelines for Embodied AI Data Standardization",
    "date": "2026-06-15",
    "tags": ["数据标准", "数据管线", "自动驾驶"],
    "summary": "自动驾驶领域已经走过具身智能正在走的路...",
    "summaryEn": "The autonomous driving industry has already...",
    "contentHtml": "<h2>一、自动驾驶数据管线的成熟度</h2><p>...</p>",
    "contentHtmlEn": "<h2>1. Maturity of AV Data Pipelines</h2><p>...</p>",
    "author": "Your Name"
  }
]
```

正文直接存 HTML（不引入 Markdown 解析器，保持零依赖）。

#### 模板

新增 2 个页面模板：
- `src/pages/blog.html` — 文章列表页：卡片布局，按标签筛选，按日期排序，分页
- `src/pages/blog-detail.html` — 文章详情页：标题、日期、标签、正文、底部作者信息、返回链接

英文版对应模板在 `src/pages/en/` 下。

#### RSS

`build.cjs` 生成博客列表时同步输出 `dist/rss.xml`，方便用户用 RSS 阅读器订阅。

格式为标准 RSS 2.0，不含复杂扩展。

#### 导航入口

顶部导航栏增加 "Blog / 技术博客" 入口，中英文站各自显示对应文案。

### 内容策略

首期发布 3-5 篇种子文章（确保博客上线时不是空的），建议主题：
1. 《从自动驾驶数据管线看具身智能数据标准化》——利用你的领域优势
2. 《具身智能数据集格式全景对比》——基于现有 55 个数据集的分析
3. 《高质量机器人操作数据的判断标准》——质量标签体系的展开论述
4. 《开源具身数据生态现状与缺口》——行业趋势分析
5. 《XX 数据集深度评测》——单个数据集的详细分析模板

之后以 2-4 周一篇的节奏持续产出。

## 模块三：数据格式对比 + 标准草案

### 3a. 现有对比功能增强

在数据集对比页（`datasets.html` 中的 compare 面板）增加"数据格式"对比维度：

- **Schema 树状**：并排展示各数据集的 schema 结构，差异字段用黄色高亮
- **存储格式**：RLDS / TFDS / HDF5 / 自定义，用标签显示
- **传感器类型**：列表展示，共同项和差异项区分颜色
- **数据规模**：数据集大小、轨迹数对比

最多同时对比 4 个（沿用现有限制）。

### 3b. 格式全景图页面

新增 `src/pages/formats.html`，提供全局格式分布视图：

- 存储格式分布（饼图/柱状图，纯 CSS 实现或轻量 SVG）
- Schema 层级深度分布
- 传感器类型覆盖矩阵（行=数据集类型，列=传感器，交叉点=是否支持）
- 标准格式 vs 非标准格式统计

数据从 `datasets.json` 中的 `dataFormat` 和 `dataContent` 字段聚合统计。

### 3c. 标准草案页面

新增 `src/pages/standard-proposal.html`：

展示一版"具身智能统一数据格式"草案，包含：
- 推荐的 schema 结构：`episode → step → observation / action / info`
- 推荐的存储格式：RLDS（基于 TFDS），附带理由
- 推荐的元数据字段列表（camera intrinsics、joint states、language instruction）
- 与现有标准的关系（RLDS、LeRobot format、自定义格式的兼容映射表）

页面底部集成 giscus 评论区，收集社区反馈。

**语气定位**：专业但谦虚——"基于 55 个数据集的分析结论，不是权威标准，欢迎讨论"。

### 数据补全工作

此模块依赖高质量的数据格式元数据。现状：`dataFormat.schema` 字段在多数数据集中填写了相同模板值。需要在开发前/中补全。建议：

1. 优先补全 TOP 20 热门数据集的详细格式信息（Open X-Embodiment、BridgeData V2、RH20T 等）
2. 其余数据集使用已有信息，标注"待验证"

## 技术约束

- **零 npm 依赖**：继续使用 Node.js 原生 API 构建，不引入 Markdown 解析器、i18n 库等
- **纯静态部署**：GitHub Pages + gh-pages，不引入后端/数据库
- **GitHub Actions 自动部署**：push → build → deploy，构建时间控制在 30 秒内
- **无运行时 JS 框架**：页面交互（筛选、排序、对比）继续用原生 JavaScript

## 构建脚本改造

`build.cjs` 需要增加的逻辑：

1. **i18n 支持**：根据 `LANG` 参数选择数据文件和模板目录
2. **博客构建**：读取 `blog.json`，生成列表页 + 详情页
3. **RSS 生成**：输出 `rss.xml`
4. **格式全景图**：聚合统计数据，注入 `formats.html`
5. **标准草案**：标准草案内容从独立的 JSON 或直接硬编码在模板中

构建命令：
```bash
node build.cjs           # 一次构建，同时输出中英文站到 dist/ 和 dist/en/
```

构建脚本内部先处理中文、再处理英文，一次运行完成，确保部署一致性。
